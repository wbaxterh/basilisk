## At a glance

x402 reuses the original HTTP `402 Payment Required` status code as a real
settlement protocol. An agent's request flow looks like this:

```
agent → GET /v1/prices/MIN
server → 402  { price: 1000, receiver: "addr1...", nonce: "..." }
agent → signs Cardano tx paying 1000 lovelace, retries with X-402-Payment header
server → 200 { price, volume, change24h }
```

No accounts. No keys. No SaaS contract. Just a signed transaction per request.
