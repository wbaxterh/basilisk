Section 1

1. What "MVP" Means

The PRD defines four phases. This plan covers MVP = Phase 0 (Foundations) + Phase 1 (MVP Web) — roughly Months 0–5. The goal: a usable web product at rough parity with TapTools' core analytics + portfolio surfaces, plus a public Data API beta.

MVP Success Bar

A trader can connect/watch a wallet and see correct net worth + P&L; open any token and see a correct live price + chart + recent trades; screen the market; look up any wallet; set an alert that fires; and a developer can pull token/market/wallet data from a documented REST API with a free key.

Section 2

2. Assumptions & Team

  • Team: 2–4 engineers. Lean split: 1 backend/data (owns ingestion + pricing — the moat), 1 full-stack (API + portfolio/wallet), 1 frontend (web), + fractional design/PM. Estimates assume ~3 FTE.
  • Estimates are in story points (Fibonacci; 1 pt ≈ ½ day of one engineer incl. test + review). Planning numbers, not commitments.
  • Stack (PRD §7.4): Next.js (web) · NestJS/TypeScript + Go for perf-critical ingestion · Postgres + TimescaleDB + Redis · Cardano data via Demeter.run (§6) · TradingView lightweight-charts.
  • Data strategy (§6): start managed (Demeter + Blockfrost free fallback) rather than self-hosting a node — removes ~1–3 weeks of db-sync ops from the critical path.

Section 4

4. Backlog: Epics → Stories → Tasks

Story IDs are US-<epic>.<n>. Each carries acceptance criteria (AC) and tasks. Priority: Must Should Nice.

EPIC-0 — Data Infrastructure (the moat)

Without correct, fresh prices nothing else matters. This epic is the critical path.

US-0.1 Connect to Cardano chain data Must8 pts

As the platform, I need reliable access to live + historical Cardano chain data so all downstream services have a source of truth.

AC: Read chain tip, stream new blocks/txs, query history; failover if primary degrades.

  • Provision Demeter.run; stand up Ogmios + Kupo + db-sync (scale-to-zero) ports.
  • Wire Blockfrost free tier (50k req/day) as REST fallback/cross-check.
  • Thin internal chain-data client lib wrapping both, with health checks + circuit breaker.
  • Define which addresses/policies Kupo indexes (DEX pool scripts + on-demand wallets).
US-0.2 Ingest the chain into our pipeline Must13 pts

As the data layer, I want to tail new blocks and persist relevant events for a real-time + queryable stream.

AC: New DEX swaps appear within the <5s freshness target; pipeline survives restarts and resumes.

  • Ogmios/Oura chain-follower (Go/TS) → normalized event stream → Redis pub/sub.
  • Idempotent checkpointing (resume from last slot) + Cardano rollback handling.
  • Persist raw swap/pool events to TimescaleDB.
US-0.3 DEX adapters (decode pools & swaps) Must34 pts

As the pricing engine, I need each DEX's pools and swaps decoded into reserves + trade events to derive price and liquidity.

AC: Per DEX, reserves and swaps reconstructed accurately and validated vs the DEX's own UI for sample tokens.

  • Adapter framework + shared interface (getPools, decodeSwap, getReserves).
  • Minswap (do first — highest volume), SundaeSwap, WingRiders, MuesliSwap, VyFinance adapters.
  • Spike: evaluate Charli3 Dendrite (OSS SDK already unifying these DEXs) to accelerate or replace hand-rolled adapters — could cut this story substantially.
  • Per-adapter conformance tests + golden fixtures; alarm on decode failures (protocols upgrade scripts).
US-0.4 Pricing engine: VWAP + OHLCV Must21 pts

As a user, I want an accurate single price per token so charts, portfolio, and screeners all agree.

AC: Volume-weighted across pools (ADA base) → USD via oracle; OHLCV for 1m/5m/15m/1h/4h/1d; matches manual spot-check.

  • VWAP aggregation across pools; ADA/USD oracle (cross-check ≥2 sources).
  • OHLCV candle builder → TimescaleDB continuous aggregates; backfill top-N tokens via db-sync.
  • Live candle updates → Redis pub/sub for chart/portfolio consumers.
US-0.5 Quality / spam filter Must8 pts

As a user, I don't want scam/dust tokens polluting screeners and prices.

AC: Tokens below liquidity/volume thresholds flagged "low quality" and excluded from default lists.

  • Configurable thresholds (min liq, min vol, age); outlier handling for thin pools; quarantine list + admin override.
US-0.6 Token metadata service Must8 pts

As a user, I want token names, tickers, logos, decimals, supply, and policy IDs.

AC: Each token resolves to standard metadata (CIP-25/68 + registry); missing data degrades gracefully.

  • Resolve from token registry + on-chain metadata; cache in Postgres; supply/FDV/mcap computation.
US-0.7 Reorg / data-correctness safeguards Must10 pts
  • Rollback-aware writes; reconciliation job cross-checking prices vs Blockfrost; data-freshness SLA monitor (p95 dashboard); staleness alerting.

Section 5

5. Definition of Done

A story is done when: code reviewed + merged · unit/integration tests pass · meets AC · instrumented (logs/metrics) · no secrets in code · docs/changelog updated · deployed to staging and smoke-tested.

Data stories carry an extra gate

Any story touching prices, balances, or P&L additionally requires a correctness cross-check against an external source (Blockfrost / the DEX's own UI) before it's accepted.

Section 6

6. Cardano Data Layer & the "Dedalus" Question

Finding up front

"Dedalus" is not a real Cardano data-infra product. The name almost certainly refers to Demeter.run (by TxPipe) — the actual managed Cardano infrastructure PaaS (phonetically De-meter ≈ De-dalus). The only similar real thing is Daedalus, IOG's desktop full-node wallet — an end-user app, irrelevant to our data layer. The intended product is Demeter, and it's a good fit.

Recommended Path (MVP)

Demeter (Ogmios + Kupo + db-sync) as primary, Blockfrost free as fallback, Charli3 Dendrite to accelerate DEX parsing. Real-time prices via Ogmios/Oura + Dendrite-parsed pools; historical OHLCV via db-sync scale-to-zero SQL; wallet/portfolio UTxOs via Kupo. Estimated Cardano-data cost at MVP scale: ~$20–40/month. Migration trigger: once historical-aggregation load climbs or we need guaranteed real-time latency at scale → move node + db-sync in-house on Hetzner (~$60–120/mo fixed).

How It Scales

Time-series DB is the steepest curve — budget it as the #1 scaling cost. Keep storage/CDN on Cloudflare (R2 zero-egress) to avoid the AWS egress trap. Trajectory: MVP ~$50–225/mo → 10k users ~$500–1,500/mo → 50k+ users w/ heavy time-series $2,000–6,000+/mo.
